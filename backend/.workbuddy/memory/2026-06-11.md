# 2026-06-11 工作日志

## 适配 users 表新增 email/openid 字段

数据库 users 表新增了 email 和 openid 字段（之前 openid 用 phone 字段代替存储），完成以下改动：

### 修改文件清单
1. **Users.java** — 新增 `email`/`openid` 字段及 getter/setter
2. **UsersMapper.xml** — resultMap、Base_Column_List、insert、insertSelective、updateByExample、updateByExampleSelective、updateByPrimaryKey、updateByPrimaryKeySelective 全部加入 email/openid 两列
3. **UsersExample.java** — 新增 andEmailEqualTo/andEmailLike/andOpenidEqualTo 等查询方法
4. **AuthServiceImpl.java** — 核心逻辑改为 `andOpenidEqualTo(code)` 查询，新用户注册用 `user.setOpenid(code)` 代替原来的 `user.setPhone(code)`
5. **UserServiceImpl.java** — getProfile 输出新增 email 字段
6. **UserProfileVO.java** — 新增 email 字段
7. **LoginVO.java** — UserInfoVO 内嵌类新增 email 字段

### 逻辑变化
- 旧：code（微信临时码）→ 存入 phone 字段
- 新：code → 存入 openid 字段；phone 字段留给真实手机号

## 15:09 — AuthServiceImpl 最终简化版

登录逻辑改为最简单：
- `code` 就是 userId → `Long.parseLong(code)` → `selectByPrimaryKey(userId)`
- 不存在则报错（不自动注册，用户由数据库预建）
- token 存入 openid 字段
- phone 字段完全不动（留给真实手机号）
- 移除了 UsersExample、自动注册、member_ext 新建逻辑
